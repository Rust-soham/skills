# Foundations

## Curriculum Invariant

Design a sequence of leverage-ordered model transitions, not a topic list.
Each step should create present capability while lowering the cost of later
learning through reused foundations, better questions, feedback, or artifacts.

## Mission Shapes the Graph

`unblock-build`, `abstraction-untangle`, `research-map`,
`layered-capability`, and `transfer-synthesis` require different graph width,
step size, evidence, and return conditions. They share the same sequencing
philosophy: select the next reachable transition with the greatest future
leverage.

## Recursive Pareto

Use recursive Pareto as a selection heuristic, not a growth claim. After each
verified unlock, re-evaluate the remaining search space and choose the next
small reachable transition with the largest expected leverage for the current
mission.

## Learner Ownership

The path selects a threshold; `learn-deeply` lets the learner cross it. Design
steps that require prediction, encounter evidence, expose a failed assumption,
and verify reconstruction through application or transfer. Do not substitute
agent output, topical coverage, or quiz activity for a model transition.

## Agent Function

Use the agent to remove incidental friction and arrange evidence, not to own
the learner's judgment. Treat increasing ability to generate discriminating
questions, assess output, and choose the next experiment as progress.

## Claim Discipline

Treat "exponential" as an objective of multiplicative reuse and growing
learning leverage. Name the causal mechanism for each step; do not promise a
numerical growth law.
