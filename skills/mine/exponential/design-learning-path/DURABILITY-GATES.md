# Durability Gates

Use a durability gate after a major transition: one reused by later steps,
needed for independent success, or likely to create an illusion of fluency.
Do not impose one after every small step or delay an urgent artifact return.

## Purpose

The initial gate establishes that the learner can repair a model now. A
durability gate asks whether the model remains available after the original
prompt, wording, and scaffold no longer carry the answer.

## Choose a Gate

Use the smallest test that establishes the needed kind of durability:

- **Retrieval:** ask the learner to reconstruct the invariant, trace, or
  decision rule without notes or the original example.
- **Near transfer:** change the surface details while preserving the same
  structure; require a prediction or implementation.
- **Boundary transfer:** change one relevant condition and ask where the old
  rule stops applying.
- **Integration:** use the model inside a later artifact, review, or final
  boss without naming the earlier lesson.

Schedule it after a context shift, at the next meaningful encounter, or before
a successor depends on the model. Use elapsed time only when the mission has a
real retention requirement; avoid arbitrary streaks or quota pressure.

## Run It

1. Remove the prior hint, trace, or worked example.
2. Ask for a prediction, reconstruction, choice, or implementation.
3. Change only enough context to make memorized wording insufficient.
4. Judge the model, not exact phrasing or a lucky output.
5. If it holds, record durability or transfer evidence and raise later task
   complexity.
6. If it fails, locate the decayed assumption and reopen only the smallest
   repair path. Do not replay the entire lesson or treat the miss as failure.

## Path Placement

Attach a gate to the later step that consumes the earlier foundation whenever
possible. This makes retention serve the mission instead of becoming detached
maintenance. Use a dedicated gate only when a critical model has no natural
later use before independent success.

## Evidence

Update the existing learning record with the gate type, changed context,
learner response, assistance used, and result. A durable check is evidence of
availability or a new diagnosis; it is not a score, streak, or label.
