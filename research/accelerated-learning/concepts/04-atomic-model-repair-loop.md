# Atomic Model-Repair Loop

**Status:** Core

## Core Claim

The primary learning unit is one evidence-backed change to one important model,
not a lesson consumed, answer produced, or topic completed.

## Canonical Loop

`elicit -> predict -> collide -> observe -> diagnose -> repair -> compress ->
verify`

1. Elicit the current model through action or explanation.
2. Require a prediction.
3. Run a targeted case.
4. Compare prediction with observation.
5. Identify the failed assumption.
6. Introduce the minimum useful repair.
7. Compress the repair into an invariant and boundary.
8. Verify it in a novel case.

## Required Output

- Old model
- Externalized or named representation when structure would otherwise remain
  vague
- Prediction or attempted action
- Observation
- Failed assumption
- Repaired invariant
- Boundary where the old model remains useful
- Compressed rule
- Verification result

## Boundaries

- Change one load-bearing assumption at a time.
- Do not persist an unverified repair.
- Preserve superseded models and the conditions they failed instead of erasing
  the transition.
- Do not confuse an inspectable correct answer with an inspectable model delta.
