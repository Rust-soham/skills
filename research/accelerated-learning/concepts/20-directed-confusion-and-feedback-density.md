# Directed Confusion and Feedback Density

**Status:** Core

## Core Claim

The model-building phase cannot be removed. It can be compressed by directing
the learner toward the right uncertainty and increasing the density of clear,
causally interpretable feedback.

The target is speed of correct realization, not raw task speed, content volume,
or the number of attempts.

## Compression Levers

- Reduce randomness by locating the most likely load-bearing gap.
- Remove material irrelevant to the current model transition.
- Change one meaningful variable at a time.
- Make prediction, outcome, and failed assumption easy to compare.
- Shorten the loop from attempt to inspectable evidence.
- Detect practice-target misalignment before increasing repetition.
- Return quickly to another prediction after a repair.

## Compounding vs Resetting

A feedback loop compounds when each failure becomes structure:

`learn -> build -> fail -> diagnose -> refine -> reuse`

It resets when context is abandoned before the error is compressed:

`consume -> copy -> get stuck -> switch -> repeat`

AI can accelerate either loop. Interaction speed is therefore not evidence of
learning acceleration; the question is whether each iteration improves the
connected model and the learner's ability to generate the next experiment.

## Quality Test

A denser loop is useful only when:

- Each iteration produces evidence about the target model.
- The learner still predicts, diagnoses, and repairs.
- Errors become more specific rather than merely more frequent.
- Repetition is aimed at the actual bottleneck.
- Later iterations require less undirected search.

## Boundaries

- Do not eliminate confusion before it exposes the missing model.
- More cycles against the wrong target compound error, not learning.
- Immediate feedback is harmful when it supplies closure before interpretation.
- Breakthrough feelings are evidence of reorganization, not an optimization
  target.
