# Chosen Friction

**Status:** Core

## Core Claim

Preserve resistance that changes the model the learner wants to own. Remove
resistance that merely consumes attention through retrieval, syntax,
boilerplate, setup, formatting, or already-understood mechanics.

## Mechanism

- The learner declares the capability or cognition they intend to develop.
- The agent classifies current friction as target or incidental.
- Target friction stays learner-owned and evidence-producing.
- Incidental friction is retrieved, automated, delegated, or isolated away.
- The classification changes as capability changes.

## Diagnostic Questions

- What should the learner be able to predict or decide independently?
- Does this difficulty expose that model or only delay contact with it?
- If AI removes this step, which capability stops developing?
- If the step is preserved, what evidence will it produce?

## Boundaries

- Difficulty is not automatically productive.
- A mechanical question should be answered directly when it is non-target
  friction.
- Repeated departure from chosen friction can be a signal, not proof, of
  avoidance.
