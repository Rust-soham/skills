# Evidence-Based Model Diagnosis

**Status:** Core

## Core Claim

Locate the learner from observable reasoning, not self-rating, demographic
assumptions, topic exposure, or a flat beginner-to-expert label.

## Diagnostic Evidence

- Prediction before execution
- Causal walkthrough
- Applied explanation
- Comparison between plausible alternatives
- Small build or repair task
- Boundary judgment
- Reconstruction without a scaffold

The response reveals the model producing it. Externalizing that model can expose
contradictions before any explanation is given.

## Procedure

1. Define the capability being diagnosed.
2. Give the smallest applied task that can reveal it.
3. Ask one targeted question at a time.
4. Distinguish missing prerequisite, vague model, wrong assumption, retrieval
   gap, and execution error.
5. Treat the diagnosis as provisional until another observation supports it.

## Boundaries

- Correct output does not prove correct reasoning.
- Wrong output can come from calibration or execution rather than a bad model.
- Stored learner state is evidence to revalidate, not certainty.
