# Curriculum as Leverage Architecture

**Status:** Core

## Core Claim

A curriculum is a graph of transformations ordered by what accelerates future
learning. It is not a topic inventory or a universal easy-to-hard sequence.

The target is a step-by-step path whose order creates the steepest defensible
compounding curve. Each threshold should increase current competency and
increase the rate at which later competency can be acquired.

## Curriculum Sequence

1. Autopsy the domain and expert perception.
2. Identify or mine primitives, invariants, prerequisites, limiting models, and
   candidate threshold changes.
3. Establish only the substrate needed for meaningful action.
4. Choose the highest-leverage reachable threshold.
5. Practice under causally relevant use conditions.
6. Recontextualize earlier knowledge through the new model.
7. Integrate, transfer, and produce durable evidence.

## Selection Rule

Leverage chooses the destination. Current demonstrated capability chooses the
size and scaffolding of the next step.

Each module should answer:

- What capability does this unlock?
- Which earlier foundations does it consume?
- Which future learning friction does it remove?
- Which later transformations become cheaper?
- What prior knowledge changes meaning afterward?
- What artifact and gate prove generative use?
- How does the environment make the correct next action easiest?

Treat "exponential" as multiplicative reuse and increasing leverage, not a
numerical guarantee. A threshold edge is incomplete until its causal
compounding mechanism is named.

## Boundaries

- Do not include every conventional prerequisite.
- Do not claim every stage is a threshold.
- Do not include inert foundations that no later step consumes.
- Keep domain structure separate from learner evidence.
- Treat mined thresholds as hypotheses until domain grounding and generative
  learner evidence support them.
