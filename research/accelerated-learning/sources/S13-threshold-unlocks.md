### Source Authority Within the Transcript

technical claims are not independently verified and are not the research

The user rejects an expert-level explanation, asks for a developer analogy,
then corrects that analogy by naming strong algebra as the better anchor

[552-620](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L552)).

They later specify the desired sequence: algebra first, then code, then bare

([754-818](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L754),
[914-996](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L914)).

`SOURCE`: The closest familiar domain is not always the obvious demographic
analogy. Anchor selection must use demonstrated knowledge and learner feedback.

### Unlock as Removal of a Limiting Model

The user identifies a kind of click that changes the view of many connected
ideas. The transcript describes it as dropping a subtly limiting mental picture
rather than merely acquiring a fact
([1000-1038](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1000)).

Unlock shape:

1. A concept is usable under a working model.
2. The model contains a hidden assumption that creates recurring friction.
3. A more minimal or structural perspective becomes visible.
4. The old clutter dissolves and multiple prior facts compress together.

([1029-1038](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1029))

`REFINEMENT`: The old model may be locally valid rather than simply false. The
upgrade loosens its scope and exposes a more generative invariant.

### Layered Compression and Unlearning

The user rejects binary understanding and proposes repeated application,
limiting cases, getting stuck, and unlearning bad assumptions
([1052-1056](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1052)).

The resulting layers are:

- Mechanical use
- A second perspective that destabilizes hidden assumptions
- A wall the current model cannot explain
- Identification and replacement of the specific bad assumption

([1072-1081](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1072))

Useful friction is the falsification of a current model, not confusion in
general
([1085-1091](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1085)).

Unlearning is difficult because limiting assumptions are often almost right,
useful at the current layer, and invisible
([1095-1111](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1095)).

### Unlock Sequence

The user proposes simulating clicks through curated exercises ordered by
understanding layer
([1115-1125](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1115)).

The sequence is:

1. Establish a baseline case the current model handles.
2. Present a near-boundary case that creates specific friction.
3. Present a limiting case that breaks the model.
4. Offer or elicit the minimum reframe resolving the contradiction.
5. Verify with cases only the upgraded model can solve.

([1129-1141](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1129))

The monoid example has the learner generalize several concrete operations, fail
on an unusual type, then invent the minimum interface before receiving its name
([1145-1157](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1145)).

`REFINEMENT`: Explanation timing matters. A reframe that would bounce off before
the contradiction can compress the model after the learner has accumulated the
right friction.

### Diagnosis and Ordering

The hard problem is not generating exercises in the abstract. It is identifying
which assumption the particular learner carries and which exercise breaks only
that assumption
([1161-1165](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1161)).

The proposed loop is:

`answer -> infer producing belief -> locate belief transition -> target shallow
wrong node -> generate controlled collision -> minimal reframe -> verify`

([1181-1221](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1181)).

`SOURCE`: A good collision is easy enough for the old model to attempt
confidently and specific enough to fail for one diagnostic reason
([1187-1194](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1187)).

### Belief-Transition Graph

The transcript proposes an assumption graph whose nodes contain wrong and
repaired beliefs, while edges contain the minimal reframe needed for the
transition
([1223-1237](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1223),
[1319-1340](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1319)).

Suggested node data includes:

- Wrong belief
- Correct or more generative belief
- Minimal reframe
- Diagnostic question
- Verification question

([1344-1368](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1344))

Real learner answers should add previously unknown misconceptions to the graph
([1372-1378](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1372)).

`QUESTION`: A misconception graph is useful, but a strict DAG may be too simple.
Beliefs can be context-dependent, mutually reinforcing, or regress under load.

### Dual-Role Tutor

The user proposes that the agent simultaneously model:

- The learner's local perspective, to meet them where they are.
- The expert destination and dependency structure, to move them forward.

([1274-1305](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1274))

Expert-only instruction can target the right concept at the wrong level.
Learner-only simulation can mirror confusion without finding the shortest
transition
([1309-1315](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1309)).

`SYNTHESIS`: These are reasoning functions, not necessarily separate agents.
One function reconstructs the learner model from evidence; another evaluates
the shortest valid path toward the domain model.

### Questions as Diagnostic Probes

The user insists that placement should come from questions rather than assumed
learner levels
([1382-1395](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1382)).

A diagnostic probe asks what model produces an answer, not merely whether a fact
is known. One answer may reveal several belief nodes
([1398-1420](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1398)).

The opening should therefore use a minimal concrete task that produces a rich
signal rather than "what do you know about X?"
([1424-1439](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1424)).

`SOURCE`: The diagnostic phase is already instruction because articulating a
belief can expose its inconsistency
([1455-1461](../../../transcripts/the%20caption%20is%2C%20_Illustrating%20a%20tabulation%20in%20a%20do.md#L1455)).

### Durable State Suggested by the Source

- Demonstrated anchors and rejected analogies
- Current working model and scope
- Hidden or suspected limiting assumption
- Baseline, boundary, breaking, reframe, and verification tasks
- Belief-transition graph
- Evidence supporting learner placement
- New misconceptions discovered from real answers
- Unlock history and connected concepts

### Signals

- Learner uses the current model confidently on a baseline
- One near-boundary case creates a specific mismatch
- Learner can name the limiting assumption
- Minimal reframe compresses several facts at once
- Upgraded model handles novel verification cases
- Learner invents the abstraction before receiving its label
- One diagnostic response reveals multiple beliefs
- A new answer updates the misconception graph

### Failure Modes

- Treating understanding as binary
- Leading with the abstraction's name or expert definition
- Random exercise ordering
- Breaking several assumptions simultaneously
- Treating every old model as wholly false
- Assuming learner level from demographics or topic labels
- Static misconception taxonomies
- Expert explanations that ignore the learner's current representation
- Learner simulation without an expert destination

### Refinements and Tensions

`REFINEMENT`: S01 and S02's threshold concepts gain an exercise-level mechanism:
establish, strain, break, minimally reframe, and verify.

`REFINEMENT`: S09's concept graph should record belief transitions and evidence,
not only topic strengths or prerequisites.

`QUESTION`: "Start with zero assumption" is impossible in a literal sense. The
safer rule is to treat prior models as hypotheses and require learner evidence
before acting on them.

`QUESTION`: Some misconceptions collapse through articulation; others require
execution or external evidence. The system must choose probe modality based on
the kind of claim being tested.
